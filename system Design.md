System Context DiagramSince this is an offline-capable client-side app, the system design is streamlined:User Device (Mobile/Desktop Browser) <--> Vercel Edge Network (Static Hosting)All compute, state management, and rendering happen directly within the user's browser DOM.2. Data FlowInput: User inputs a string -> React updates options state.Render: React recalculates $360 / options.length$ -> updates DOM with new CSS conic-gradient.Action: User clicks Spin -> React sets isSpinning to true -> calculates target degree -> updates rotation state.Animation: Browser engine handles hardware-accelerated CSS transform rotation over a set duration (e.g., 4000ms).Resolution: Transition ends -> React calculates winner index based on final modulo angle -> updates winner state -> Result Modal renders.3. Future Scalability (Phase 2)If you decide to add features like "Shared Group Sessions" (where everyone sees the wheel spinning on their own phones simultaneously):Real-time Infrastructure: Introduce a Node.js server using WebSockets (or a service like Pusher/Supabase).State Syncing: The host clicks "Spin", the server generates the random target angle and broadcasts it to all connected clients. All clients execute the frontend CSS animation simultaneously.
