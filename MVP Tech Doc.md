State ManagementThe app relies on a few critical pieces of React state (useState):options (Array of Strings): e.g., ['Saugata', 'Rahul', 'Amit'].rotation (Number): The current absolute rotation of the wheel in degrees.isSpinning (Boolean): Locks the input fields and spin button while the animation runs.winner (String | null): Stores the winning option to trigger the result modal.2. Core Algorithms & LogicSlice Calculation: Each slice's angle is calculated dynamically: $Slice Angle = \frac{360}{N}$ (where $N$ is the number of options).Spin Logic:When the spin button is clicked, generate a random target angle and add multiple full rotations to ensure a long spin.$New Rotation = Current Rotation + (360 \times 5) + Random(0, 360)$Winner Calculation:Once the CSS transition ends (using onTransitionEnd event), calculate which slice is at the 0-degree mark (top of the wheel).Normalize the rotation: $Final Angle = New Rotation \pmod{360}$Account for the offset to find the top slice.Map the angle back to the options array index to declare the winner.3. PWA ConfigurationInclude a manifest.json with app icons, short name, and display: "standalone".Implement a basic Service Worker to cache the index.html and static assets, allowing the app to load even if the user drops offline in a restaurant.
